# MLSA-K8S: Multi-Layer Security Architecture for Cloud-Native Systems on Kubernetes

> Capstone Project 2 · CMU-CS 451 · International School, Duy Tan University  
> Team C2NE.03 · Supervisor: Eng. Binh, Van Nguyen · Version 3.0 Final

## 📋 Overview

MLSA-K8S implements a **7-layer defense-in-depth security architecture** on Google Cloud's **GKE Standard** cluster with **automated attack evaluation**. Deployed on ZONAL infrastructure (asia-southeast1-b) optimized for the $300 free credit budget.

### 7 Security Layers

| Layer | Name | Primary Tools | Status |
|---|---|---|---|
| **L1** | Infrastructure Security | GKE Shielded Nodes, COS image, node hardening | ✅ |
| **L2** | Control Plane Security | RBAC (least-privilege), OPA/Gatekeeper, Audit Logging | ✅ |
| **L3** | Identity & Secrets | Workload Identity, cert-manager, Istio mTLS STRICT | ✅ |
| **L4** | Network Segmentation | NetworkPolicy deny-all + explicit allow, Calico | ✅ |
| **L5** | Supply Chain Security | Trivy Operator, Cosign, Binary Authorization | ✅ |
| **L6** | Workload & Runtime | PSA labels, seccomp, AppArmor, Falco | ✅ |
| **L7** | Observability & Exposure | NGINX Ingress + TLS, Prometheus, Grafana, Loki | ✅ |

## 🚀 Quick Start

### Prerequisites

```bash
# macOS / Linux
brew install kubectl helm terraform cosign trivy
brew install google-cloud-sdk

# Windows (PowerShell Admin)
winget install Kubernetes.kubectl Helm.Helm Hashicorp.Terraform Google.CloudSDK

# Authenticate with GCP
gcloud auth login
gcloud auth application-default login
gcloud config set project mlsa-k8s-capstone
gcloud services enable container.googleapis.com binaryauthorization.googleapis.com
```

### Setup (3 Commands)

```bash
# 1. Create infrastructure (GKE cluster + node pools)
make setup

# 2. Bootstrap Helm tools (cert-manager, Istio, Gatekeeper, Falco, etc)
make bootstrap

# 3. Deploy all 7 security layers
make deploy-all
```

## ⚡ Deployment Guide

### Step 0: Install Local Tools

```bash
# Verify installations
kubectl version --client
helm version
terraform version
istioctl version

# Add istioctl to PATH (macOS example)
export PATH=$HOME/istio-X.Y.Z/bin:$PATH
```

### Step 1: Create Infrastructure (Terraform)

```bash
# Initialize GCS bucket (run ONCE, before terraform init)
bash infrastructure/terraform/scripts/create-tfstate-bucket.sh

# Deploy GKE cluster + node pools
cd infrastructure/terraform
terraform init
terraform plan -var="admin_cidr=$(curl -s ifconfig.me)/32" -out=tfplan
terraform apply tfplan

# Get kubeconfig (ZONAL syntax - important!)
gcloud container clusters get-credentials mlsa-k8s-cluster \
  --zone asia-southeast1-b \
  --project mlsa-k8s-capstone
```

### Step 2: Bootstrap Helm Tools

```bash
# Create namespaces + install cert-manager + Istio + Gatekeeper
bash infrastructure/scripts/bootstrap.sh

# Verify installations
kubectl get deployments -n cert-manager
kubectl get pods -n istio-system
kubectl get pods -n gatekeeper-system
```

### Step 3: Deploy 7 Layers

```bash
# Deploy all layers in order (dependencies respected)
make deploy-all

# Or deploy individually:
make deploy-l1  # Infrastructure
make deploy-l2  # RBAC + Gatekeeper
make deploy-l3  # Identity + mTLS
make deploy-l4  # NetworkPolicy
make deploy-l5  # Supply Chain
make deploy-l6  # Workload Protection
make deploy-l7  # Observability
make deploy-app # Demo apps
```

## 📊 Architecture Decisions

| Decision | Rationale | Impact |
|---|---|---|
| **ZONAL (asia-southeast1-b)** | Regional 3× more expensive | Saves $600/month vs $313 full cost |
| **e2-standard-4 for system pool** | Istio + Gatekeeper + Falco = ~2GB overhead | Prevents OOMKill on e2-standard-2 |
| **Evaluation pool autoscale min=0** | Scenarios rarely run | ~$4-10/month vs $97/month |
| **Terraform for audit logging** | GKE managed, no kubectl apply audit-policy.yaml | Centralized GCP Cloud Logging |
| **Helm for tools** | Version-controlled, repeatable | All tools versionable and upgradeable |
| **kubectl apply for custom manifests** | IaC separation, K8s-native | Fine-grained RBAC/NetworkPolicy control |

## 🏗️ Project Structure

```
Kubernetes manifests: kubectl apply
├── Namespaces with PSA labels
├── RBAC: ServiceAccounts, ClusterRoles, RoleBindings
├── Gatekeeper: ConstraintTemplate + Constraint YAML
├── Istio: PeerAuthentication, AuthorizationPolicy
├── NetworkPolicy: deny-all + allow rules per layer
├── Falco: custom-rules.yaml for runtime monitoring

Terraform IaC: terraform apply
├── GKE cluster with ZONAL zone + logging_config + binary_auth
├── 3 node pools (system e2-standard-4, workload e2-standard-2, evaluation autoscale)
├── VPC + subnets + firewall rules
├── Workload Identity bindings (KSA ↔ GSA)

Helm tools: helm install
├── cert-manager (TLS certificate management)
├── Istio (service mesh + mTLS)
├── Gatekeeper (admission control)
├── Falco (runtime security)
├── Trivy Operator (vulnerability scanning)
├── Prometheus + Grafana + Loki (observability stack)
├── NGINX Ingress (L7 exposure)
```
